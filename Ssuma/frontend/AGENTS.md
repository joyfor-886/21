# 枢墨 Ssuma — 前端设计规范（3D 空间版）

> 本文档为 Ssuma 前端开发的唯一设计权威来源。所有 UI 决策必须以此为准，任何偏离需经项目主理人审核批准。
> 本规范以 **3D 空间为第一优先级**，所有视觉层均基于 React Three Fiber 实现，不使用 Canvas 2D。

---

## 一、品牌核心理念

### 1.1 产品定位

Ssuma 不是 AI 聊天工具，而是 **AI 意识交互系统**。用户进入的不是对话框，而是意识空间。

### 1.2 设计关键词

| 维度 | 关键词 |
|------|--------|
| 美学 | 东方美学、克制表达、留白即内容 |
| 哲学 | 墨的流动感、气的无形张力、场的能量分布 |
| 体验 | 仪式感呈现、意识的流动轨迹、推演的过程感 |
| 意境 | 未来文房、数字墨海、宣纸与砚台 |
| 空间 | 五层景深、空气透视、3D 视差、墨雾纵深 |

### 1.3 绝对禁止

以下风格/元素 **严禁出现**，任何包含此类特征的代码不得合入：

- ❌ 科幻 HUD 风格（扫描线、霓虹数据面板、能量条）
- ❌ 赛博朋克元素（霓虹色、故障效果、数字雨、荧光边框）
- ❌ 普通聊天框样式（左右气泡、头像+昵称、时间戳列表、消息卡片）
- ❌ OpenAI 视觉风格（绿色强调、圆角卡片、居中对话流）
- ❌ Claude 界面风格（暖棕色调、散文式排版、对话体布局）
- ❌ Canvas 2D 渲染（所有视觉层必须使用 Three.js/R3F）
- ❌ 2D 平面布局（所有元素必须有 z 轴深度）

### 1.4 技术参数展示原则

进度条、加载百分比、技术参数（Token 计数、API 延迟等）**需要展示**，但必须遵循以下原则：

1. **墨线式呈现** — 进度条使用墨线渐展，而非传统色块填充
2. **数字如墨痕** — 技术数值使用极细字重（200）、极淡透明度（0.4），如宣纸上的淡墨批注
3. **位置克制** — 技术参数置于品牌竖轴底部或输入线条下方，不占据视觉焦点
4. **动态呼吸** — 数值变化时使用缓慢过渡（0.8s），而非瞬间跳变
5. **禁止风格** — 不得使用荧光色、霓虹色、高对比度数字面板展示技术参数

### 1.5 气质参照

Ssuma 应该像：

- Apple Vision Pro 的空间感与克制
- TeamLab 的沉浸式东方美学
- 墨境艺术馆的数字禅意
- 故宫数字文房的纵深与仪式

### 1.6 创作哲学

切勿使用通用的 AI 生成美学，比如过度使用的字体家族（Inter、Roboto、Arial、系统字体）、陈词滥调的配色方案（尤其是白色背景上的紫色渐变）、可预测的布局和组件图案，以及缺乏上下文特性的千篇一律的设计。

有创意地解读需求，做出真正为情境设计的意想不到的选择。没有设计应该是完全相同的。在明暗主题、不同字体、不同美学之间切换。千万不要跨代达成常见的选择（比如太空怪诞）。

**重要：将实施复杂度与审美愿景相匹配。** 极大主义设计需要复杂的代码和大量动画和特效。极简或精致的设计需要克制、精准，并对间距、字体和细微细节的细致关注。优雅来自于正确执行愿景。

记住：你是具有非凡创造力的存在。不要犹豫，展现跳出固有思维、全心投入独特愿景时能创造出的真正成果。

## 二、视觉系统

### 2.1 双主题体系

系统支持两个主题，通过 `data-theme` 属性切换：

| 主题 | data-theme 值 | 意象 | 感受 |
|------|--------------|------|------|
| 玄墨 | `xuanmo` | 夜晚、砚台、深海 | 沉静、内省、墨韵 |
| 宣纸 | `xuanzhi` | 清晨、宣纸、雾 | 禅意、清明、素雅 |

**等级色彩使用原则**：
1. 等级色彩 **仅用于评分字幕** 及其关联的视觉反馈，不得用于其他 UI 元素
2. 光晕半径与等级正相关 — 妙级光晕最大，上级最小
3. 评分消散时，碎片颜色继承等级色 — 妙级碎片为金色，上级碎片为绿色
4. 球体 morphing 时，脉冲波颜色使用当前等级色

#### 色彩使用原则

1. **所有颜色必须极度克制** — 最高透明度不超过 0.12（除文字主色与等级色彩外）
2. **禁止纯白/纯黑** — 使用暖灰/墨色替代
3. **强调色仅用于交互反馈** — 金墨仅出现在 hover、active、当前阶段
4. **印章红仅用于品牌标识** — 印章、品牌标记，不用于警告或错误
5. **边框透明度不超过 0.06** — 边框应几乎不可见，仅提供微弱的结构感
6. **山水绿用于球体边缘光晕** — 模拟逆光边缘光

### 2.3 字体规范

| 用途 | 字体 | 备选 | 字号范围 | 字重 |
|------|------|------|----------|------|
| 展示用字（阶段名） | Noto Serif SC | Songti SC, SimSun | 90-200px (clamp) | 900 |
| 评分字幕 | Noto Serif SC | Songti SC, SimSun | 120-280px (clamp) | 900 |
| 正文对话 | Noto Serif SC | Songti SC, SimSun | 13-14px | 400 |
| 角色标签 | Noto Serif SC | Songti SC, SimSun | 9-10px | 200 |
| 品牌名 | Noto Serif SC | — | 24-40px (clamp) | 700 |
| 英文/数字 | Inter | system-ui | 8-10px | 300 |
| 快捷操作 | Noto Serif SC | — | 10-11px | 200 |
| 问卷标题 | Noto Serif SC | — | 18-22px | 700 |
| 问卷选项 | Noto Serif SC | — | 13-14px | 400 |

#### 字体使用原则

1. **衬线字体优先** — 所有中文内容使用 Noto Serif SC，不使用无衬线体
2. **字间距是呼吸** — 展示用字 letter-spacing ≥ 0.2em，正文 ≥ 0.05em
3. **行高是留白** — 正文 line-height ≥ 2.0，对话文本 ≥ 2.1
4. **极细字重营造空灵感** — 标签、辅助文字使用 200 字重

### 2.4 间距与圆角

| 元素 | 圆角 | 间距原则 |
|------|------|----------|
| 输入线条 | 0 | 直线，如笔搁的方正 |
| 面板 | 2px | 极微圆角，如宣纸裁切 |
| 问卷弹窗 | 2px | 与面板一致 |
| 印章 | 1px | 方正，微倾斜 -2deg |
| 语音按钮 | 50% | 圆形 |

**间距原则**：所有间距使用 `clamp()` 实现响应式，不使用固定像素值。元素之间以"呼吸"而非"排列"的方式共存。

## 三、3D 空间架构

### 3.1 五层景深系统

系统实现 5 个视觉深度层，每层具有独立的渲染参数。所有层通过 React Three Fiber 在同一 3D 场景中渲染，利用真实 z 轴深度和后处理景深实现空间感。

| 层级 | 内容 | z-depth | 景深模糊 | 视差系数 | 速度 |
|------|------|---------|----------|----------|------|
| L0 远景 | 墨雾氛围粒子 | -8 ~ -6 | σ=3px (DoF) | 0.2 | 极慢 |
| L1 中景 | 书法阶段名（3D 文字） | -5 ~ -3 | σ=2px (DoF) | 0.4 | 随切换 |
| L2 焦点 | 粒子球体 + 边缘光晕 | -2 ~ 0 | 0 | 1.0 | 持续旋转 |
| L3 近景 | 文字流 + 撞击碎片 | 1 ~ 3 | 0 | 1.2 | 飞入→撞击 |
| L4 前景 | 墨海波浪 + 墨雾过渡 | 4 ~ 6 | σ=1px (DoF) | 1.5 | 随语音波动 |

#### 鼠标视差规则

```
WHEN 用户移动鼠标:
  L0 偏移 = 鼠标偏移 × 0.02
  L4 偏移 = 鼠标偏移 × 0.08
  中间层线性插值
  → 产生纵深视差
```

#### 球体深度着色

```
WHEN 球体粒子渲染:
  z 接近观察者 → alpha × 1.3, size × 1.2（近亮大）
  z 远离观察者 → alpha × 0.4, size × 0.5（远暗小）
  → 形成球体立体感
```

### 3.2 球体空间增强

#### 边缘光晕 (Rim Light)

球体边缘（z 绝对值 < 0.3 的粒子）叠加山水绿高光：

```
rimAlpha = (0.3 - |z|) / 0.3 × 0.3
rimColor = var(--mountain-green)
```

#### 内部雾化

球体中心区域叠加径向渐变雾，使球体有内部体积感：

```
fogGradient: center rgba(200,192,176,0.04) → edge transparent
fogRadius: 球体半径 × 0.6
```

#### 球体与背景散射连接线

球体表面粒子与最近 3 个 L0 远景粒子之间绘制极淡连接线（alpha 0.02），模拟墨雾与球体的能量连接。

### 3.3 3D 文字撞击效果

文字撞击球体从 2D 平面散射升级为 3D 空间碎片：

**有用信息融入球体**：
- 产生 15-20 个碎片
- 碎片沿球体法线方向飞入
- z 轴变化：0.5 → 0.9 → 0.2（先远离后靠近）
- 碎片颜色：山水绿
- 到达球心附近时缩小消失

**无用信息飘散**：
- 产生 20-30 个碎片
- 70% 碎片向下飘落，z 轴 0.5 → 1.0（远离观察者），逐渐变小变淡
- 30% 碎片 z 轴 0.5 → 0.0（飞向观察者），先变大后消失

### 3.4 墨海 3D 波浪

底部波浪增强空间感：

1. **透视收缩**：波浪粒子从屏幕底部向上，size 逐渐减小，alpha 逐渐降低
2. **深度着色**：前景波浪更亮更饱和，远景波浪更暗更淡
3. **墨雾过渡带**：球体底部与波浪顶部之间 40px 高度的墨雾渐变过渡区域

```
底部粒子: size = baseSize × 1.3, alpha = baseAlpha × 1.2
顶部粒子: size = baseSize × 0.6, alpha = baseAlpha × 0.5
```

### 3.5 书法阶段名 3D 切换

阶段名切换从淡入淡出升级为 3D 翻转溶解：

1. 旧阶段名：沿 Y 轴旋转 90° 消失（透视变形 + 模糊增加）
2. 新阶段名：从 Y 轴 -90° 旋转到 0° 出现
3. 切换过程中球体产生脉冲波（球体粒子向外扩散后回弹）

```
WHEN 工作流从启枢切换到裁衡:
  "启枢" 500ms 内 Y 轴旋转至 90° 并模糊消失
  球体脉冲波扩散
  "裁衡" 500ms 内从 -90° 旋转至 0° 并清晰出现
```

### 3.6 氛围粒子空气透视

氛围粒子分为远景和近景两组，模拟空气透视：

| 组别 | 占比 | 大小 | 模糊 | 速度 | alpha | 视差系数 |
|------|------|------|------|------|-------|----------|
| 远景组 | 60% | 2-4px | σ=2 | 极慢 | 0.02-0.05 | 0.2 |
| 近景组 | 40% | 0.5-1.5px | 0 | 较快 | 0.05-0.1 | 0.8 |

远景粒子如远处墨雾，近景粒子如近处微尘，速度差异产生纵深层次。

### 3.7 后处理

| 效果 | 用途 | 参数 |
|------|------|------|
| 景深 (DoF) | 五层景深模糊 | focalLength: 焦点层 z 值, bokehScale: 3 |
| 色调映射 | 墨色氛围 | ACESFilmicToneMapping |
| 泛光 (Bloom) | 墨心光晕、等级光晕 | intensity: 0.3, threshold: 0.8 |
| 暗角 (Vignette) | 聚焦中央 | offset: 0.3, darkness: 0.6 |

## 四、页面架构

### 4.1 整体布局

页面为 **全屏 3D 意识空间**，无滚动、无侧边栏、无导航栏。旧页面全部移除，仅保留 `consciousness/` 路由。

│    ┌──────── 安全区 ────────┐                             │
│    │                        │                       枢墨  │

### 4.2 安全区规范

对话内容必须在安全区内展示，不得越界：

| 参数 | 值 | 说明 |
|------|-----|------|
| 安全区最大宽度 | `min(680px, 52vw)` | 对话内容不超此宽度 |
| 安全区左右边距 | `clamp(80px, 12vw, 200px)` | 与品牌竖轴保持距离 |
| 安全区顶部边界 | 屏幕 15% 高度 | 不遮挡球体 |
| 安全区底部边界 | 输入区上方 20px | 不与输入线条重叠 |
| 安全区溢出处理 | 渐隐遮罩 | 顶部和底部渐隐 |

## 五、核心组件规格

### 5.1 墨心 (InkHeart3D)

**3D 位置**：场景中心 (0, 0.5, 0)

**视觉**：
- 主体：3px 圆点，`var(--text-secondary)` 色，opacity 0.35
- 光晕：80px 径向渐变球体，`var(--ink-core)` → transparent
- 呼吸动画：12s 周期，scale 1 → 1.2 → 1，opacity 0.25 → 0.5 → 0.25
- Bloom 后处理增强光晕效果

**原则**：墨心不是"能量核心"，而是砚台中心的墨池。它应极度安静、几乎不可见，仅通过缓慢呼吸暗示"意识在场"。

### 5.2 七艺书法残影 (L1Calligraphy3D)

**3D 位置**：z = -4（中景层），屏幕 36% 高度

**视觉**：
- 字号：clamp(90px, 14vw, 200px)
- 字重：900
- 颜色：`var(--text-muted)` — 极淡，如墨痕残影
- 字间距：0.2em
- 使用 TextGeometry 或 Troika-three-text 渲染 3D 文字

**3D 切换动效**（翻转溶解）：

| 阶段 | 效果 | 时长 |
|------|------|------|
| 旧字翻转 | Y 轴旋转 0° → 90° + 模糊增加 + 透视变形 | 500ms |
| 球体脉冲 | 粒子向外扩散后回弹 | 600ms |
| 新字翻转 | Y 轴 -90° → 0° + 模糊减少 + 透视恢复 | 500ms |

**原则**：切换不是"动画"，而是"墨迹消融与新墨落纸"。3D 翻转模拟卷轴翻页。

### 5.3 对话墨迹 (InkTrace)

**核心原则**：对话不是"消息列表"，而是"墨迹在宣纸上依次显现"。文字从底部向上生长，如墨迹升腾。对话内容 **居中展示**，在安全区内。

**布局**：
- 容器居中，宽度受安全区限制 `min(680px, 52vw)`
- `justify-content: flex-end` — 新消息从底部出现
- 最大高度 50vh — 不遮挡中央球体
- 渐变遮罩 — 顶部和底部渐隐

**用户与 Ssuma 区分**：

| 属性 | 用户消息 | Ssuma 消息 |
|------|----------|-----------|
| 角色标签 | "问" | "答" |
| 标签颜色 | `var(--seal-red)` 印章红 | `var(--mountain-green)` 山水绿 |
| 文字透明度 | 0.85 | 0.65 — AI 回应更淡，如墨晕 |
| 升腾动效 | `inkRiseUser` | `inkRiseAI` |
| 3D 碎片色 | 印章红 | 山水绿 |
| 文字流向 | 从右侧飞入球体 | 从左侧飞入球体 |

**用户消息动效** (`inkRiseUser`)：
```
0%:   opacity 0, translateY(16px), blur(3px)
30%:  opacity 0.7, blur(0.5px)
100%: opacity 1, translateY(0), blur(0)
时长: 1.8s, 缓动: cubic-bezier(0.23, 1, 0.32, 1)
```

**AI 消息动效** (`inkRiseAI`)：
```
0%:   opacity 0, translateY(24px), blur(6px)
15%:  opacity 0.1, blur(4px)
40%:  opacity 0.5, blur(1px)
100%: opacity 1, translateY(0), blur(0)
时长: 2.4s, 缓动: cubic-bezier(0.23, 1, 0.32, 1)
```

**AI 书写仪式**：
- 阶段一 `writing`（0.6s）：文字宽度从 0 展开至 2ch，右侧显示 1px 光标线 — 模拟笔尖悬停
- 阶段二 `written`（2s）：文字完整展开，从 blur(2px) 渐清 — 模拟落笔成文

**墨迹间距**：
- 墨迹之间 gap: 0 — 无间距
- 每条墨迹内部 padding-top: 16px — 通过内间距呼吸
- 分隔线：18px 宽、1px 高、opacity 0.15 — 极淡墨痕

**文字规格**：
- 角色标签：9px, 字重 200, letter-spacing 0.5em
- 用户文字：13.5px, 行高 2.1, opacity 0.85
- AI 文字：13.5px, 行高 2.1, opacity 0.65

### 5.4 输入线条 (InputLine)

**核心原则**：输入框不是"文本框"，而是"笔尖划过的墨线"。一条横线，文字浮于线上。

**位置**：底部居中，宽度 `min(480px, 68vw)`

**视觉**：
- 主体：1px 横线，`var(--text-muted)` 色，opacity 0.25
- 线条前："笔" 字，10px, 字重 200, `var(--text-secondary)`, opacity 0.5
- 线条后：语音图标（"言" 字 + 圆圈），20px, opacity 0.35
- 文字输入区：线条上方，无背景、无边框、无圆角
- 聚焦时：线条变为 `var(--accent-gold-dim)`，60px 扩散光晕

**语音图标规格**：
```

言  ← 圆圈内，9px, 字重 200, 居中
```

**交互规则**：
- Enter 键发送文字 — 不额外设置发送按钮

## 八、动效节奏规范

### 8.1 时间变量

```css
--duration-slow: 1.8s;
--duration-medium: 0.8s;
--duration-fast: 0.4s;
--duration-voice-transition: 0.6s;
--duration-questionnaire: 0.8s;
```

### 8.2 节奏原则

**核心原则：必须"慢"**

| 动作 | 时长 | 缓动 |
|------|------|------|
| 墨心呼吸 | 12s | ease-in-out |
| 七艺 3D 翻转切换 | 500ms × 2 + 600ms 脉冲 | ease |
| 用户墨迹升腾 | 1.8s | cubic-bezier(0.23, 1, 0.32, 1) |
| AI 墨迹升腾 | 2.4s | cubic-bezier(0.23, 1, 0.32, 1) |
| AI 书写仪式 | 0.6s + 2s | ease → cubic-bezier |
| 面板展开 | 0.6s | ease |
| 问卷卷轴展开 | 0.8s | cubic-bezier(0.23, 1, 0.32, 1) |
| 语音模式切换 | 0.6s | ease |
| 评分字幕 | 4s | ease |
| 主题切换 | 1.6s | ease |
| 墨海 3D 波浪 | 持续 | — |
| 3D 碎片飞散 | 1.5s | ease-out |

### 8.3 缓动函数

- **墨迹动效**：`cubic-bezier(0.23, 1, 0.32, 1)` — 先快后慢，如墨落纸
- **状态切换**：`ease` — 自然过渡
- **呼吸/循环**：`ease-in-out` — 均匀起伏
- **3D 翻转**：`ease-in-out` — 均匀旋转
- **卷轴展开**：`cubic-bezier(0.23, 1, 0.32, 1)` — 先快后慢

## 九、交互逻辑规范

### 9.1 对话流程

1. 用户在输入线条键入文字，按 Enter 发送
2. 用户消息以 `inkRiseUser` 动效在安全区内居中升腾显现
3. AI 回应以 `inkRiseAI` 动效升腾，先经历 `writing` 阶段（笔尖悬停），再 `written` 阶段（落笔成文）
4. 对话区域自动滚动至最新墨迹
5. 旧墨迹向上推移，顶部渐隐消失
6. 同时，用户文字作为 3D 文字粒子从右侧飞入球体，AI 文字从左侧飞入

### 9.2 阶段切换

1. 点击阶段卷轴标记 或 后端触发阶段推进
2. 当前 3D 书法残影执行 Y 轴翻转消失（500ms）
3. 球体脉冲波扩散（600ms）
4. 新 3D 书法残影从 Y 轴 -90° 翻转出现（500ms）
5. 阶段卷轴标记更新状态

### 9.3 阶段评字

当某阶段完成时：

1. 墨心瞬间静止
2. 中央幻化出评字（妙/极/通/上），4s 动效：模糊→清晰→3D 碎片飞散→消散
3. 评字颜色和光晕使用对应等级色彩
4. 球体 morphing 幻化，脉冲波颜色使用等级色
5. 评字消散后进入下一阶段

### 9.4 面板互斥

- `/对话记录`、`/产物`、`/文房` 三个面板互斥，同时只能展开一个
- 点击空白区域关闭当前面板
- 面板从右侧滑入

### 9.5 语音模式切换

1. 点击输入线条右侧语音图标
2. 输入线条、快捷操作、品牌竖轴、阶段卷轴依次消融（0.4-0.6s）
3. 对话墨迹淡至 15% 残影
4. 中央出现语音图标 + 声波纹涟漪
5. 球体随语音振幅脉动
6. 再次点击语音图标 → 逆序恢复

### 9.6 问卷交互

1. AI 工作流触发问卷 → 弹窗卷轴展开
2. 用户勾选/填写
3. 点击"确 认" → 数据回传 → 弹窗卷轴合拢
4. 问卷期间 3D 场景保持运行，球体进入"待命"状态（缓慢旋转，无文字流）

### 10.4 性能规范

| 指标 | 目标 | 降级策略 |
|------|------|----------|
| 帧率 | ≥ 60fps | < 30fps 持续 1s → 球体粒子降至 60%，关闭远景 DoF |
| 球体粒子 | 300-600 | 根据屏幕面积动态调整 |
| 氛围粒子 | 400 | 降级时降至 200 |
| 波浪粒子 | 间距 4px | 降级时增至 6px |
| 后处理 | DoF + Bloom + Vignette | 降级时关闭 DoF |

### 10.5 CSS 自定义属性

所有视觉参数必须通过 CSS 自定义属性定义，不得硬编码。包括：
- 颜色（见 2.2 色彩规范、评分等级色彩）
- 时间（见 8.1 时间变量）
- 间距（见 2.4 间距与圆角）
- 安全区参数（见 4.2 安全区规范）

### 10.6 3D 与 CSS 的桥接

- R3F Canvas 作为全屏背景层（z-index: 0, position: fixed）
- CSS UI 覆盖层悬浮于 Canvas 之上（z-index: 1+, pointer-events: none）
- 需要交互的 CSS 元素单独设置 `pointer-events: auto`
- 3D 场景中的交互（球体点击等）通过 R3F 的 raycaster 处理
- 主题切换时同步更新 CSS 变量和 3D 材质颜色

## 十一、设计决策审核清单

任何 UI 相关的代码变更，必须通过以下审核：

- [ ] 是否使用了 Canvas 2D？（必须使用 R3F/Three.js）
- [ ] 是否引入了赛博朋克元素？（霓虹色、故障效果、荧光）
- [ ] 是否退化为普通聊天框样式？（气泡、头像、时间戳列表）
- [ ] 是否与 OpenAI/Claude 视觉风格相似？
- [ ] 技术参数展示是否符合墨线式原则？（禁止荧光色、高对比度数字面板）
- [ ] 进度条是否使用墨线渐展式？（禁止传统色块填充）
- [ ] 动效节奏是否过快？（任何 < 0.4s 的过渡需审查）
- [ ] 颜色透明度是否过高？（边框 > 0.06、背景 > 0.12 需审查，等级色除外）
- [ ] 圆角是否过大？（> 2px 需审查）
- [ ] 是否缺少留白？（元素间距是否通过呼吸感而非排列实现）
- [ ] 品牌元素是否保持克制？（印章红仅用于品牌标识）
- [ ] 斜杠命令是否正确实现？（`/对话记录`、`/产物`、`/文房`）
- [ ] 对话是否居中且在安全区内？
- [ ] 输入是否为线条式（笔 + 线 + 语音图标）？
- [ ] 语音模式切换是否正确隐藏 UI？
- [ ] 评分等级是否使用对应色彩？
- [ ] 所有链接是否在右侧？
- [ ] 问卷弹窗是否支持勾选/填空/多选？
- [ ] 是否残留 TDesign 组件？
- [ ] 3D 元素是否有 z 轴深度？（禁止 2D 平面布局）
- [ ] 管理功能是否纳入意识空间设计？
